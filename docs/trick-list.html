System Definicji Akcji (ActionDefs)

Zakładka Akcje na arkuszu Active Effect pozwala zdefiniować deklaratywne opisy akcji walki wręcz — zamiast wpisywać surowe obiekty JS bezpośrednio w skryptach getMeleeActions. Definicje są przechowywane w effect.system.actionDefs[] i edytowane przez UI.

Pola definicji akcji

Triggery melee: onMeleeHit / onMeleeBlock / onMeleeTakeover

Te trzy triggery są pełnoprawnymi wyzwalaczami systemu — widocznymi w dropdownie selektora wyzwalacza w zakładce Skrypty efektu, tak jak getMeleeActions czy opposedAttacker. Odpytują obu uczestników po każdym rozstrzygnięciu segmentu duela, wewnątrz applyDuelBatch() — po wyliczeniu outcome, ale przed zapisem stanu i przed aplikacją obrażeń przez GM.

Pełny reference melee combat (wszystkie triggery, timing, wzorce): melee-info.html

Różnica od opposedAttacker/opposedDefender: tamte triggery odpytują tylko jedną stronę (każdą osobno) i nie mają MeleeActionContext — nie wiedzą jaka sztuczka była użyta. onMeleeHit/Block/Takeover odpytują obu uczestników i dostarczają pełny kontekst akcji (w tym args.action.id ze zdefiniowanej sztuczki).

Powiązanie z getMeleeActions — dwa podejścia do efektów przy trafieniu

Gdy akcja zdefiniowana w getMeleeActions trafi, masz dwie drogi obsługi efektów. Można je łączyć na tym samym efekcie:

PodejścieGdzie piszesz kodKiedy używać
onHitScript w zakładce Akcje Pole onHitScript w edytorze akcji Efekt ściśle związany z tą akcją. Prosta aplikacja AE na cel. Potrzebujesz immediateOnHit (odpala w trakcie duela).
Trigger onMeleeHit w zakładce Skrypty Nowy skrypt efektu z wyzwalaczem onMeleeHit Logika zależy od kontekstu (tier, sukcesy obu stron, poprzednie trafienia). Chcesz reagować na każde trafienie niezależnie od akcji. Potrzebujesz args.context lub args.hits.
Zalecany przepis na sztuczki z efektem: Zdefiniuj akcję w zakładce Akcje (name + damage). W zakładce Skrypty dodaj drugi skrypt z wyzwalaczem onMeleeHit — sprawdzaj args.action?.id i aplikuj efekt. Nie potrzebujesz onHitScript w edytorze akcji.

Przykłady użycia

onMeleeHit Zastosuj efekt na cel gdy konkretna sztuczka trafia — zalecany wzorzec
// Trigger: onMeleeHit — skrypt na tym samym efekcie co akcja w zakładce Akcje
if (!args.isAttacker) return; // tylko atakujący aplikuje efekt
if (args.action?.id !== "PASTE_ACTIONDEF_ID_HERE") return; // filtruj po ID akcji z zakładki Akcje
const target = args.context.defender.actor;
if (!target) return;
const tmpl = this.effect?.parent?.effects?.getName("Ogłuszony");
if (tmpl && !this.isEffectActive(tmpl, target)) {
  await target.createEmbeddedDocuments("ActiveEffect", [tmpl.convertToApplied()]);
}
onMeleeHit Efekt na siebie gdy trafiasz (dowolna akcja, min. 2 sukcesy)
// Trigger: onMeleeHit — atakujący dostaje buff gdy trafi z przewagą
if (!args.isAttacker) return;
if (args.attackerSuccesses - args.defenderSuccesses < 2) return;
const tmpl = this.effect?.parent?.effects?.getName("Furia");
if (tmpl) await args.actor.createEmbeddedDocuments("ActiveEffect", [tmpl.convertToApplied()]);
onMeleeTakeover Reakcja na przejęcie inicjatywy — tylko obrońca
// Trigger: onMeleeTakeover — obrońca przejął inicjatywę
if (args.isAttacker) return; // tylko strona, która przejmuje (obrońca)
const advantage = args.defenderSuccesses - args.attackerSuccesses;
ui.notifications.info(`Przejęcie inicjatywy! Przewaga: +${advantage}`);
onMeleeBlock Powiadom czat gdy obrońca blokuje z przewagą
// Trigger: onMeleeBlock — obaj mają sukcesy, wynik remis
if (args.isAttacker) return; // tylko obrońca reaguje
if (args.defenderSuccesses < args.attackerSuccesses) return;
ChatMessage.create({ content: `${args.actor.name} blokuje z ${args.defenderSuccesses} sukcesami!` });
Ważne: Trigger odpala się dwa razy — raz dla atakującego, raz dla obrońcy. Zawsze sprawdzaj args.isAttacker, żeby nie wykonywać logiki dwa razy.
state jest read-only: Triggery onMeleeHit/Block/Takeover nie mogą trwale modyfikować stanu duelu. Do zmian aktywnych w trakcie trwania walki użyj onHitScript z immediateOnHit: true. Do zmian przed segmentem użyj preOpposedAttacker / preOpposedDefender.
