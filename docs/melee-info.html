Cykl życia walki wręcz

Poniższa oś czasu przedstawia dokładną kolejność, w jakiej uruchamiane są triggery walki wręcz w trakcie rundy.

Rozpoczęcie tury (Turn start)
1 meleeUpdate (phase: "turn-start")
Następuje reset puli kości uczestników oraz wyczyszczenie efektów tymczasowych w przestrzeni nazw encounter._effects.
2 preMeleePool (3k20 Roll)
Gracz wykonuje rzut 3k20. Na tym etapie skrypty mogą modyfikować parametry attackTargetSnapshot oraz defenseTargetSnapshot uczestnika.
3 meleeUpdate (phase: "pool-ready")
Obie pule kości (ataku i obrony) zostają ostatecznie zatwierdzone i przygotowane do podziału na segmenty.
Przebieg segmentu (Per segment — powtarzany dla każdego z 3 segmentów)
4 meleeUpdate (phase: "attack-committed")
Atakujący deklaruje liczbę kości przeznaczonych na dany segment.
5 preOpposedAttacker
Uruchamiane po stronie atakującego tuż przed kalkulacją sukcesów. Pozwala na modyfikację trudności difficultyTarget.
6 preOpposedDefender
Uruchamiane po stronie obrońcy tuż przed kalkulacją sukcesów. Pozwala na modyfikację trudności difficultyTarget oraz ustawienie blockDamageShift.
7 System computes successes
System automatycznie oblicza liczbę sukcesów i określa wynik segmentu (trafienie / remis-blok / przejęcie inicjatywy). Jeśli sztuczka posiada natychmiastowy skrypt (immediateOnHit: true), zostanie on wykonany w tym miejscu.
8 onMeleeHit / onMeleeBlock / onMeleeTakeover
Odpala się dla OBU stron (osobne wywołanie dla atakującego i obrońcy) w zależności od wyznaczonego wyniku segmentu.
9 opposedAttacker
Uruchamiane wyłącznie po stronie atakującego. Pozwala na nadpisanie ostatecznego typu wyniku resultType oraz modyfikację blockDamageShift.
10 opposedDefender
Uruchamiane wyłącznie po stronie obrońcy. Pozwala na zmodyfikowanie wartości defender.blockDamageShift.
11 meleeUpdate (phase: "exchange-resolved")
Kalkulacje dla danego segmentu zostały zakończone i zapisane w stanie pojedynku.
12 meleeUpdate (phase: "segment-advance")
Następuje przejście indeksu segmentu do przodu.
Zakończenie tury (Turn end)
13 meleeUpdate (phase: "turn-end")
Uruchamiane pod koniec tury pojedynku, tuż przed zresetowaniem całego stanu kości i przygotowaniem nowej rundy.
Aplikacja obrażeń (Damage apply — gdy GM klika "Nałóż obrażenia")
14 onHitScript (deferred)
Dla każdego odnotowanego trafienia segmentu, system odpala odłożone skrypty trafień ze sztuczek (zapisane w trickOnHitScripts).
15 preTakeDamage & takeDamage
Wykonywane indywidualnie dla każdej rany nakładanej na docelowego aktora.

MeleeActionContext

Obiekt MeleeActionContext jest tworzony metodą statyczną MeleeActionContext.fromSegment() wewnątrz applyDuelBatch(). Jest przekazywany jako parametr args.context w triggerach onMeleeHit, onMeleeBlock oraz onMeleeTakeover. Parametr args.action to bezpośredni skrót do args.context.action.

action
object|null
Deskryptor sztuczki użytej w tym segmencie lub null dla zwykłego ataku.
action.id
string
Unikalne ID sztuczki.
action.name
string
Nazwa wyświetlana sztuczki.
action.damage
string|null
Typ obrażeń sztuczki (np. "L", "2C", "sD"), wyciągnięty z rekordu hitEntry.
action.effectIds
string[]
Tablica powiązanych ID efektów Active Effects z przedmiotu macierzystego.
action.effectTiming
string
Moment nałożenia efektów: "onHit" | "afterDamage" | "onUse".
action.effectTarget
string
Cel nałożenia efektów: "target" | "self" | "selected" | "manual".
action.sourceEffectUuid
string|null
UUID źródłowego efektu ActiveEffect.

Użycie kontekstu w skrypcie onMeleeHit
if (args.isAttacker && args.action?.id === "moja-sztuczka-id") {
  const target = args.context.defender.actor;
  const successes = args.context.attacker.successes;
  if (target && successes >= 2) {
    await this.sendMessage("Sztuczka trafia z potężną siłą!");
  }
}

Pola deskryptora akcji (getMeleeActions)

Podczas wywołania triggera getMeleeActions do tablicy args.actions można dodawać obiekty konfigurujące dynamiczne przyciski akcji walki. Poniższa tabela przedstawia wszystkie akceptowane pola takiego deskryptora.

Pole Typ Opis
id string Unikalny, stabilny identyfikator akcji (najlepiej pobrany z zakładki ActionDefs).
name string Etykieta przycisku wyświetlana na karcie pojedynku.
img string Opcjonalna ścieżka do pliku ikony graficznej przycisku.
damage string|null Typ rany: D/L/C/K (lub z prefiksem s- dla siniaków), np. "L", "2C", "sD". null oznacza brak obrażeń.
successCost number Koszt akcji w sukcesach. 0 oznacza akcję standalone (osobny przycisk); wartość > 0 aktywuje automatyczną kolejkę sukcesów przed zadeklarowaniem kości.
minDice / maxDice number Minimalna i maksymalna liczba kości dozwolona przy deklaracji (tylko akcje standalone).
onHitScript string Kod JavaScript uruchamiany w momencie trafienia. Zastępuje standardowy mechanizm aplikacji rany, chyba że określono inaczej.
immediateOnHit boolean Jeśli true, skrypt onHitScript odpala się natychmiast przy rozstrzyganiu segmentu (nie czeka na naciśnięcie przycisku aplikacji obrażeń).
effectIds string[] Lista identyfikatorów Active Effects z przedmiotu macierzystego, które zostaną automatycznie zaaplikowane przy trafieniu.
effectTiming string Określa moment nałożenia efektów: "onHit" | "onUse" | "afterDamage".
effectTarget string Cel dla aplikacji efektów: "target" | "self" | "selected" | "manual".
applyEffectsAutomatically boolean Gdy false, automatyczna generacja kodu aplikacji efektów z effectIds zostaje wyłączona.
resolverKind string Mechanizm rozstrzygania: "opposedMelee" (standardowy test walki) lub "directApply" (bezpośrednie nałożenie bez rzutu oporu).
gmNote string Podpowiedź / notatka widoczna wyłącznie dla Mistrza Gry po najechaniu kursorem myszy na przycisk akcji.

Wzorce i przepisy

Poniżej zebrano typowe i sprawdzone fragmenty skryptów implementujących najczęstsze mechaniki sztuczek w walkach wręcz.

getMeleeActions Wzorzec 1: Deklaratywna akcja z effectIds
// Zastosuj AE "Ogłuszony" na cel gdy sztuczka trafia (bez pisania onHitScript)
args.actions.push({
  id: "ogłuszenie-id",
  name: "Ogłuszenie",
  damage: null,
  successCost: 2,
  effectIds: ["<id-efektu-ogłuszony>"],
  effectTiming: "onHit",
  effectTarget: "target"
});
getMeleeActions Wzorzec 2: Referencja przez ID z zakładki Akcje
// Referencja do gotowej konfiguracji zapisanej w edytorze ActionDefs
args.actions.push("PASTE_ACTIONDEF_ID_HERE");
getMeleeActions Wzorzec 3: Nadpisanie pola w runtime
// Zmiana kosztu w sukcesach w zależności od zaistnienia wcześniejszego trafienia
const hasPrior = args.state.hits?.some(h => h.trickId === "boa-capture");
args.actions.push({
  ...args.lookupActionDef("PASTE_ACTIONDEF_ID_HERE"),
  successCost: hasPrior ? 2 : 3
});
onMeleeHit Wzorzec 4: Zastosuj efekt gdy konkretna sztuczka trafia
// Wykrycie trafienia naszą sztuczką i nałożenie Active Effect
if (!args.isAttacker) return;
if (args.action?.id !== "moja-sztuczka-id") return;
const target = args.context.defender.actor;
if (!target) return;
const tmpl = this.effect?.parent?.effects?.getName("Ogłuszony");
if (tmpl && !this.isEffectActive(tmpl, target)) {
  await target.createEmbeddedDocuments("ActiveEffect", [tmpl.convertToApplied()]);
}
preOpposedDefender Wzorzec 5: Garda (przesunięcie obrażeń przy bloku)
// Zmniejszenie stopnia otrzymywanych ran po udanym bloku kośćmi obrony
if (!this.isActiveForMelee(args)) return;
args.participant.blockDamageShift = (args.participant.blockDamageShift ?? 0) - 1;
meleeUpdate Wzorzec 6: Efekt per tura / Wiatrak
// Przykład: Wiatrak — odzyskanie inicjatywy w fazie segment-advance
if (args.phase === "segment-advance") {
  if (args.encounter._effects?.wiatrakReclaim !== args.participantId) return;
  args.encounter.turnState.initiativeOwnerId = args.participantId;
  delete args.encounter._effects.wiatrakReclaim;
} else if (args.phase === "turn-end") {
  if (!this.isActiveForMelee(args)) return;
  await this.sendMessage("<b>Wiatrak:</b> +20% kary ze zmęczenia.");
  this.deactivateForMelee(args);
}

Kiedy użyć którego triggera

Poniższa tabela decyzyjna ułatwia dobór odpowiedniego punktu wejścia dla Twojego skryptu w zależności od zamierzonego efektu mechanicznego.

Zadanie / Cel biznesowy Wskazany trigger skryptowy
Zmień PT rzutu przed walką dialog
Zmień próg sukcesów w puli 3k20 preMeleePool
Zmień próg sukcesów ataku w konkretnym segmencie preOpposedAttacker
Zmień próg sukcesów obrony lub ustaw parametry bloku preOpposedDefender
Dynamiczne dodawanie akcji/sztuczek do karty walki getMeleeActions
Reakcja na trafienie sztuczką z pełnym kontekstem akcji onMeleeHit
Reakcja na zablokowanie ataku przez obrońcę (obie strony) onMeleeBlock
Reakcja na przejęcie inicjatywy w walce (obie strony) onMeleeTakeover
Obsługa wyniku segmentu tylko po stronie atakującego (brak kontekstu sztuczki) opposedAttacker
Obsługa wyniku segmentu tylko po stronie obrońcy (brak kontekstu sztuczki) opposedDefender
Działania na koniec tury, przechodzenie segmentów, stany inter-trigger meleeUpdate
Aplikacja natychmiastowego efektu bezpośrednio przy rozstrzyganiu starcia onHitScript (z ustawionym immediateOnHit: true)
Opóźniony efekt nakładany dopiero przy kliknięciu "Nałóż obrażenia" onHitScript (z ustawionym immediateOnHit: false)
Porównanie opposedAttacker/opposedDefender vs onMeleeHit/Block/Takeover:
  • opposedAttacker / opposedDefender: Wywoływane wyłącznie dla jednej ze stron starcia. Nie posiadają obiektu MeleeActionContext, przez co nie mają informacji, jaka konkretnie sztuczka była zadeklarowana. Umożliwiają jednak modyfikowanie parametrów takich jak resultType (nadpisanie wyniku segmentu) czy blockDamageShift.
  • onMeleeHit / onMeleeBlock / onMeleeTakeover: Wywoływane dla obu stron pojedynku (dwie osobne egzekucje skryptu). Posiadają pełny obiekt MeleeActionContext z dokładnymi informacjami o użytych sztuczkach i efektach, ale obiekt stanu pojedynku (state) jest w nich wyłącznie do odczytu (read-only).
