1. Czym są skrypty efektów?

Skrypty efektów to fragmenty kodu JavaScript przypisane do Active Effect. Działają automatycznie w określonych momentach gry (wyzwalacze) lub manualnie po kliknięciu przycisku ▶ na arkuszu efektu.

Każdy skrypt jest przechowywany jako flaga na dokumencie ActiveEffect (flags.neuroshima.scripts) i wykonywany przez klasę NeuroshimaScriptRunner. Skrypty mogą być dołączone zarówno do efektów bezpośrednio na aktorze, jak i do efektów na itemach (sztuczki, pancerze, ekwipunek).

Ważne: Skrypty z efektów na itemach są aktywne tylko gdy item jest założony/aktywny (pole system.equipped !== false). Efekty na itemach niezałożonych są pomijane.

Gdzie dodawać skrypty?

Otwórz dowolny ActiveEffect (na aktorze lub itemie) → zakładka Skrypty → Dodaj Skrypt.

Edytor skryptów oferuje:

  • Etykieta — nazwa widoczna na liście skryptów
  • Wyzwalacz — kiedy skrypt ma być uruchomiony
  • Uruchom gdy wyłączony (runIfDisabled) — checkbox; gdy zaznaczony, skrypt uruchamia się nawet jeśli efekt jest wyłączony (disabled). Przydatne np. do czyszczenia/sprzątania po wyłączeniu efektu.
  • Kod — edytor JavaScript z podświetlaniem składni (CodeMirror)

Automatyczne usuwanie itemów stworzonych przez skrypt (fromEffect)

Jeśli skrypt tworzy itemy na aktorze, możesz oflagować je tak, aby zostały automatycznie usunięte gdy efekt jest kasowany (np. po wygaśnięciu). Wystarczy ustawić flagę przy tworzeniu: 

// Trigger: immediate — stwórz item i powiąż z tym efektem
await this.actor.createEmbeddedDocuments("Item", [{
  name: "Tymczasowy bufor",
  type: "gear",
  flags: { neuroshima: { fromEffect: this.effect.id } }
}]);

Przy usunięciu efektu (deleteEffect lub wygaśnięcie) system automatycznie usuwa wszystkie itemy z flagą flags.neuroshima.fromEffect === effect.id.
Możesz też wywołać to ręcznie: await this.effect.deleteCreatedItems().
Aby pominąć auto-usuwanie przy konkretnym delete: effect.delete({ skipDeletingItems: true }).

manual Zasób "ładunki" — sprawdź i zużyj ładunek z itemu

Typowy wzorzec: sprawdź czy item ma zasób o danym kluczu, odczytaj wartość, a jeśli jest > 0 — zmniejsz o 1 i wykonaj akcję. 

// Trigger: manual — na itemie z zasobem "charges" (Ładunki, min:0, max:5)
(async () => {
  if (!this.hasItemResource(this.item, "charges")) {
    return this.notification("Brak zasobu 'charges' na tym itemie.", "error");
  }
  const current = this.getItemResource(this.item, "charges");
  if (current <= 0) {
    return this.notification("Brak ładunków!", "warn");
  }
  await this.decreaseItemResource(this.item, "charges", 1);
  await this.sendMessage(`Użyto ładunku. Pozostało: ${current - 1}.`);
})();
preWeaponShot Zasób "amunicja specjalna" — zużyj przed strzałem lub anuluj

W preWeaponShot można anulować strzał (args.cancel = true) jeśli specjalny zasób jest wyczerpany. 

// Trigger: preWeaponShot — na efekcie broni z zasobem "special_ammo"
(async () => {
  const weapon = args.weapon;  // item broni z args
  const ammo = this.getItemResource(weapon, "special_ammo");
  if (ammo === null || ammo <= 0) {
    this.notification("Brak specjalnej amunicji — strzał anulowany.", "warn");
    args.cancel = true;
    return;
  }
  await this.decreaseItemResource(weapon, "special_ammo", 1);
})();

Gdy znasz stały indeks zasobu (widoczny w kolumnie # zakładki Zasoby), możesz operować po ID. Pamiętaj: po usunięciu zasobu ze środka indeksy przepakowują się. 

// Trigger: manual — odczytaj zasób #0, wyświetl info, zresetuj do min

prepareData Bonus do cechy na podstawie wartości zasobu (synchroniczny)

prepareData jest synchroniczny — nie używaj await. Odczyt zasobu przez getItemResource jest synchroniczny i działa poprawnie. 

// Trigger: prepareData — bonus do Zręczności za każdy ładunek w zasobie "focus"
const focus = this.getItemResource(this.item, "focus");
if (focus && focus > 0) {
  args.dexterity = (args.dexterity || 0) + focus;
}

Argumenty args

Obiekt args jest zawsze przekazywany jako parametr funkcji skryptu. Jego zawartość zależy od wyzwalacza — szczegóły opisane są w sekcji 3.

Modyfikowanie args: Większość wyzwalaczy przekazuje args przez referencję. Modyfikując właściwości (np. args.stat = 15), zmieniasz wartości używane przez system. Nie zapomnij, że args.penalties to osobny obiekt — przypisz go bezpośrednio (args.penalties.mod -= 10).

Async vs Sync

Większość wyzwalaczy jest asynchronicznych — możesz używać await. Wyjątek:

prepareData i armorCalculation są synchroniczne. Nie używaj await, fetch, ani żadnych operacji asynchronicznych w tych triggerach — spowoduje to błędy lub zignorowanie zmian.

Wyzwalacz dialog jest najsilniejszym i najbardziej elastycznym typem skryptu. Skrypty tego typu pojawiają się w panelu Dialog Modifiers w oknie dialogowym rzutu i mogą być przez użytkownika włączane lub wyłączane. Wyzwalacz dialog obsługuje 4 osobne pola kodu:

4. Przykłady gotowych skryptów

Leczenie w Neuroshimie 1.5 — system nie używa tradycyjnych punktów HP. Zamiast tego każda rana to odrębny Item (type: "wound") z polem damageType (D / sD / L / sL / C / sC / K / sK). Leczenie polega na:
  • redukcji poziomu rany o N stopni (wound.reduceLevel(n))
  • usunięciu rany całkowicie (wound.heal() lub actor.healAllWounds())
  • zmniejszeniu kary rany (wound.reducePenalty(n))
W systemie Neuroshima 1.5 nie używa się bezpośredniej modyfikacji PŻ. Zamiast tego stosuj rany (applyWound) lub ich leczenie (reduceLevel, healAllWounds).
manual Propital — Wariant 1: natychmiastowo redukuje najcięższe rany (jeden skrypt)

Najprostszy wzorzec konsumabla — jeden skrypt manual bezpośrednio na itemie. Brak pośredniego efektu: zużywa sztukę i od razu redukuje każdą z najcięższych ran o 1 stopień (np. C → L, L → D, D → usunięta). 

// Trigger: manual — bezpośrednio na itemie "Propital"
(async () => {
  if (!this.item.hasQuantity || this.item.system.quantity <= 0) {
    return this.notification("Brak Propitalu!", "error");
  }
  const worst = this.actor.getWorstWounds();
  if (!worst.length) {
    return this.notification("Brak aktywnych ran.");
  }
  await this.item.reduceQuantity();
  for (const wound of worst) {
    // isMinDamage = D lub sD → leczymy (usuwamy ranę), inaczej obniżamy o 1
    if (this.isMinDamage(wound.system.damageType)) await wound.heal();
    else await wound.reduceLevel(1);   // C→L, L→D itd.
  }
  await this.sendMessage(
    `<strong>Propital</strong>: poziom najcięższych ran zredukowany.`
  );
})();
manual + startTurn Propital — Wariant 2: natychmiastowe leczenie + podtrzymane działanie przez 2 rundy

Skrypt manual (na itemie) zużywa sztukę, redukuje najcięższą ranę natychmiast, a następnie nakłada na aktora drugi efekt z czasem trwania 2 rund walki. Ten efekt posiada skrypt startTurn, który na początku każdej tury aktora redukuje o 1 stopień kolejną najcięższą ranę. Po wygaśnięciu efekt znika automatycznie.

▶ Skrypt na itemie — trigger: manual (Efekt 0 na itemie = ten skrypt; Efekt 1 na itemie = efekt podtrzymany z trigger startTurn) 

// Trigger: manual — na itemie "Propital"
// Natychmiastowo redukuje 1 ranę, potem nakłada efekt podtrzymany (2 rundy)
(async () => {
  if (!this.item.hasQuantity || this.item.system.quantity <= 0) {
    return this.notification("Brak Propitalu!", "error");
  }
  await this.item.reduceQuantity();

  // ① Natychmiastowa redukcja
  const worst = this.actor.getWorstWounds();
  for (const w of worst) {
    if (this.isMinDamage(w.system.damageType)) await w.heal();
    else await w.reduceLevel(1);
  }

5. Manualne wywoływanie — przyciski w itemach

Skrypty z triggerem manual można wywoływać bezpośrednio z listy itemów aktora. Każdy item (sztuczka, ekwipunek, modyfikacja pojazdu), który posiada efekt z co najmniej jednym skryptem manual, automatycznie wyświetla przycisk ▶ poniżej swojego wiersza na arkuszu aktora.

  1. Otwórz item (np. sztuczka, przedmiot ekwipunku)
  2. Przejdź do zakładki Effects na arkuszu itemu
  3. Utwórz lub edytuj efekt → zakładka Skrypty
  4. Dodaj skrypt z triggerem Manual
  5. Nadaj etykietę — ta etykieta pojawi się jako tekst przycisku

Changes działa przez standardowy mechanizm Foundry i jest przeliczane automatycznie przy każdej zmianie efektu. prepareData daje pełną kontrolę programistyczną i pozwala na warunkowe modyfikacje. Dla prostych bonusów liczbowych — Changes jest wygodniejszy.

9. Zasoby itemu

Zakładka Zasoby w każdej karcie itemu pozwala definiować dowolne zasoby numeryczne (np. ładunki, wytrzymałość, amunicja specjalna). Każdy zasób ma unikalny klucz tekstowy, nazwę wyświetlaną, wartość bieżącą oraz opcjonalne granice Min i Max. Zasobami można zarządzać bezpośrednio ze skryptów efektów.

Pola zasobu

PoleTypOpis
keystringUnikalny identyfikator tekstowy (używany w helperach po kluczu). Powinien być krótki i bez spacji, np. charges, fuel.
labelstringWyświetlana nazwa zasobu (widoczna w UI i w chipach podsumowania).
valuenumberAktualna wartość zasobu.
minnumberDolna granica clampowania. Domyślnie 0. Ignorowane gdy zasób ma włączone Unclamped.
maxnumberGórna granica clampowania. Wartość 0 oznacza brak górnego limitu (gdy Unclamped jest wyłączone). Ignorowane gdy zasób ma włączone Unclamped.
showInSummarybooleanGdy true, zasób pojawia się jako chip w podsumowaniu itemu na karcie aktora. Przełączany kliknięciem w badge # w zakładce Zasoby.
unclampedbooleanGdy true, silnik skryptów pomija całkowicie granice min/max przy ustawianiu wartości przez helpery. Pola Min i Max w UI są wtedy zablokowane. Przydatne do liczników nieograniczonych.

Kontrolki UI w zakładce Zasoby

  • Badge # (po lewej) — ID zasobu (indeks tablicy, 0-based). Kliknięcie przełącza showInSummary. Gdy aktywny — badge świeci na pomarańczowo, zasób pojawia się w podsumowaniu na karcie aktora.
  • ∞ (infinity) — przycisk obok kosza. Przełącza tryb Unclamped. Gdy aktywny — Min i Max są zablokowane, silnik skryptów nie clampuje wartości. Gdy nieaktywny — wartości ustawiane przez helpery są zawsze zawierane w przedziale [min, max].
  • Kosz — usuwa zasób. Pozostałe zasoby są automatycznie przepakowywane (ID zostają przenumerowane 0,1,2,…).
Uwaga o ID (indeksach): po usunięciu zasobu ze środka tablicy, indeksy pozostałych zasobów przepakowują się — zasób który miał ID=2 może stać się ID=1. Jeśli w skryptach odwołujesz się do zasobu po ID (a nie po kluczu), sprawdź indeksy po każdej zmianie struktury zasobów. Kluczem (key) jest bezpieczniejszy i preferowany sposób odwołania.

Wyświetlanie w podsumowaniu itemu

Zasoby z włączonym showInSummary pojawiają się jako chipy w sekcji Zasoby w rozwijanym podsumowaniu itemu na karcie aktora (pod Statystykami, przed Opisem). Format chipu: Nazwa: wartość / max (max pomijany gdy 0).

Helpery zacięcia broni (Jamming)

Poniższe helpery są dostępne wyłącznie w triggerach preWeaponShot, weaponJam i postWeaponShot (oraz bezpośrednio w triggerze weaponJam). Każdy z nich modyfikuje odpowiedni klucz obiektu args przekazanego przez silnik.

HelperDostępny wOpis
this.modifyJammingThreshold(args, delta) preWeaponShot Modyfikuje próg zacięcia o wartość delta. Dodatni delta = trudniej się zaciąć, ujemny = łatwiej. Wartość args.jammingThreshold nigdy nie spada poniżej 0.
this.preventWeaponJam(args) preWeaponShot Ustawia args.forceNoJam = true. Broń nie może się zaciąć w tej turze niezależnie od wyniku kości. Wyższy priorytet niż forceJam.

this.allowShotDespiteJam(args, count=1) weaponJam Ustawia args.canFireDespiteJam = true. Aktor oddaje strzał mimo zacięcia — broń nadal zostaje zacięta po strzale.

this.isStandardJam(args, min?, max?) weaponJam Zwraca true gdy wynik kości mieści się w zakresie [min, max] (włącznie) i rzut byłby sukcesem (args.wouldSucceed).
Domyślny zakres: 1–20 (każde zacięcie). Jeśli min > max, wartości są automatycznie zamieniane.
Można też przekazać samą liczbę zamiast args — wtedy sprawdza tylko zakres, bez filtrowania po sukcesie.
Przykład: isStandardJam(args, 11, 18) — standardowe i był sukces, isStandardJam(args, 19, 20) — krytyczne i był sukces.
Kolejność wykonania helperów:
preWeaponShot → sprawdzenie progu → (jeśli zacięcie) weaponJam → oddanie/pominięcie strzału → postWeaponShot.
Helper preventWeaponJam wywołany w preWeaponShot całkowicie zapobiega uruchomieniu weaponJam.

Pobieranie wartości broni w skryptach jammingu

W skryptach wyzwalaczy jammingu masz bezpośredni dostęp do args.weapon (Item broni) i args.actor (Actor strzelającego). Możesz odczytać dane broni przez standardowe API Foundry:

// Odczyt wartości zacięcia broni
const weaponJamming = args.weapon.system.jamming;     // próg zacięcia broni
const weaponName    = args.weapon.name;               // nazwa broni
const isJammed      = args.weapon.system.jammed;       // czy już zacięta

// Odczyt wartości skilla aktora (np. Rusznikarstwo)
const gunsmithLvl = args.actor.system.skills?.gunsmithing?.value ?? 0;

// Sprawdź bieżący próg (po modyfikacjach przez inne AE)
const threshold = args.jammingThreshold;              // read-only odczyt; modyfikuj przez helper
