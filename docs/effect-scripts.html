1. Czym są skrypty efektów?

Skrypty efektów to fragmenty kodu JavaScript przypisane do Active Effect. Działają automatycznie w określonych momentach gry (wyzwalacze) lub manualnie po kliknięciu przycisku ▶ na arkuszu efektu.

Każdy skrypt jest przechowywany jako flaga na dokumencie ActiveEffect (flags.neuroshima.scripts) i wykonywany przez klasę NeuroshimaScriptRunner. Skrypty mogą być dołączone zarówno do efektów bezpośrednio na aktorze, jak i do efektów na itemach (sztuczki, pancerze, ekwipunek).

Ważne: Skrypty z efektów na itemach są aktywne tylko gdy item jest założony/aktywny (pole system.equipped !== false). Efekty na itemach niezałożonych są pomijane.

Gdzie dodawać skrypty?

Otwórz dowolny ActiveEffect (na aktorze lub itemie) → zakładka Skrypty → Dodaj Skrypt.

Edytor skryptów oferuje:

  • Etykieta — nazwa widoczna na liście skryptów
  • Wyzwalacz — kiedy skrypt ma być uruchomiony
  • Kod — edytor JavaScript z podświetlaniem składni (CodeMirror)

2. Kontekst wykonania

Wewnątrz każdego skryptu masz dostęp do następujących zmiennych poprzez this (kontekst skryptu) oraz args (argumenty przekazane przez wyzwalacz):

Kontekst this

this.actorActor | nullAktor posiadający ten efekt (lub właściciel itemu z efektem)
this.itemItem | nullItem będący właścicielem efektu (null jeśli efekt bezpośrednio na aktorze)
this.effectActiveEffectTen aktywny efekt
this.labelstringEtykieta skryptu
this.triggerstringNazwa wyzwalacza który uruchomił ten skrypt
this.tokenTokenDocument | nullPierwszy token aktora na aktywnej scenie (null jeśli brak)
this.notification(msg, type?)voidPokazuje powiadomienie UI. type: "info" | "warn" | "error"
this.getChatData(merge?)objectDomyślne dane do ChatMessage.create() z speaker aktora i flavorem efektu
await this.sendMessage(content, chatData?)Promise<ChatMessage>Tworzy wiadomość czatu przypisaną do aktora

Argumenty args

Obiekt args jest zawsze przekazywany jako parametr funkcji skryptu. Jego zawartość zależy od wyzwalacza — szczegóły opisane są w sekcji 3.

Modyfikowanie args: Większość wyzwalaczy przekazuje args przez referencję. Modyfikując właściwości (np. args.stat = 15), zmieniasz wartości używane przez system. Nie zapomnij, że args.penalties to osobny obiekt — przypisz go bezpośrednio (args.penalties.mod -= 10).

Async vs Sync

Większość wyzwalaczy jest asynchronicznych — możesz używać await. Wyjątek:

prepareData i armorCalculation są synchroniczne. Nie używaj await, fetch, ani żadnych operacji asynchronicznych w tych triggerach — spowoduje to błędy lub zignorowanie zmian.

// Dla każdej lekkiej rany ustaw forcePassed = true
// Efekt np. na tricku "Hartowanie" lub "Żelazny Brzuch"
for (const wound of args.wounds) {
  const lightTypes = ["D", "sD", "L", "sL"];
  if (lightTypes.includes(wound.damageType)) {
    wound.forcePassed = true;
  }
}
applyDamage Niewrażliwość na lekkie rany — z annotation (własny tekst w raporcie czatu)

for (const wound of args.wounds) {

applyDamage Ochrona losowa — jeden rzut Xd100, każdy wynik ≤ 25 niweluje jedną ranę 
// Jeden rzut Xd100 (X = liczba ran), bez osobnych komunikatów
// Każdy die result ≤ 25 → forceSkip (rana całkowicie zniwelowana)
const wounds = args.wounds;
if (!wounds.length) return;

const threshold = 25;
const roll = await new Roll(`${wounds.length}d100`).evaluate();
const results = roll.dice[0].results.map(r => r.result);

let negated = 0;
for (let i = 0; i < wounds.length; i++) {
  if (results[i] <= threshold) {
    wounds[i].forceSkip = true;
    negated++;
  }
}

await roll.toMessage({
  speaker: ChatMessage.getSpeaker({ actor: this.actor }),
  flavor: `${this.effect.name} — Ochrona: ${negated}/${wounds.length} ran zniwelowanych (próg: ${threshold})`
});

// Zamiast całkowitej negacji, rany są "odchylone":
// forcePassed = true → rana trafia, ale odporność na ból jest automatycznie zdana
// (brak rzutu, rana rejestrowana jako sukces w teście)
const wounds = args.wounds;
if (!wounds.length) return;

const threshold = 25;
const roll = await new Roll(`${wounds.length}d100`).evaluate();
const results = roll.dice[0].results.map(r => r.result);

let deflected = 0;
for (let i = 0; i < wounds.length; i++) {
  if (results[i] <= threshold) {
    wounds[i].forcePassed = true; // auto-sukces odporności
    deflected++;

}

await roll.toMessage({
  speaker: ChatMessage.getSpeaker({ actor: this.actor }),
  flavor: `${this.effect.name} — Deflection: ${deflected}/${wounds.length} ran odchylonych`
});

5. Manualne wywoływanie — przyciski w itemach

Skrypty z triggerem manual można wywoływać bezpośrednio z listy itemów aktora. Każdy item (sztuczka, ekwipunek, modyfikacja pojazdu), który posiada efekt z co najmniej jednym skryptem manual, automatycznie wyświetla przycisk ▶ poniżej swojego wiersza na arkuszu aktora.

Jak to działa: Kliknięcie przycisku ▶ uruchamia skrypt bezpośrednio na aktorze. W skrypcie dostępne są this.actor (aktor będący właścicielem), this.item (item, na którym jest efekt) oraz this.effect.

Jak dodać przycisk do itemu

  1. Otwórz item (np. sztuczka, przedmiot ekwipunku)
  2. Przejdź do zakładki Effects na arkuszu itemu
  3. Utwórz lub edytuj efekt → zakładka Skrypty
  4. Dodaj skrypt z triggerem Manual
  5. Nadaj etykietę — ta etykieta pojawi się jako tekst przycisku
manual Wypij Wyciąg Uzdrawiający — leczy aktora, zużywa 1 szt. przedmiotu 
// Trigger: manual
// Etykieta: "Wypij"
// Na przedmiocie "Wyciąg Uzdrawiający" (gear, quantity > 0)
(async () => {
  if (!this.item.system.quantity) {
    return this.notification("Brak wyciągu!", "error");
  }

  // Zmniejsz ilość o 1
  await this.item.update({ "system.quantity": this.item.system.quantity - 1 });

  // Rzut leczenia (2d4 + bonus Budowy)
  const actor = [...game.user.targets][0]?.actor ?? this.actor;
  const tb = actor.system.attributes.constitution;
  const roll = await new Roll(`2d4 + ${tb}`).evaluate();

  const msg = await roll.toMessage({
    flavor: `<strong>${this.item.name}</strong>: Używa mikstury...`,
    speaker: ChatMessage.getSpeaker({ actor: actor })
  });

  // Czekaj na animację kości (Dice So Nice)
  if (game.dice3d) await game.dice3d.waitFor3DAnimationByMessageID(msg.id);

  // Ulecz aktora
  await actor.update({ "system.hp.value": Math.min(actor.system.hp.value + roll.total, actor.system.hp.max) });
})();
manual + deleteEffect Zapalony papieros — aplikuje efekt toksyny, usuwa go po wygaśnięciu 
// Skrypt 1 — trigger: manual, etykieta: "Zapal"
// Aplikuje drugi efekt z tego itemu na aktora
const sourceItem = this.effect.parent; // item = papieros
if (sourceItem?.effects?.size > 1) {
  const toxinEffect = sourceItem.effects.contents[1];
  await this.actor.applyEffect({ effectUuids: [toxinEffect.uuid] });

}

// Skrypt 2 — trigger: deleteEffect

await this.effect.parent?.delete?.();
this.notification("Papieros wypalony.");

WłaściwośćTypOpis

6. Zakładka Changes — modyfikacje atrybutów

Zakładka Changes pozwala modyfikować dane aktora przez standardowy mechanizm Foundry (bez pisania kodu). Każda zmiana to:

PoleOpis
Klucz (Key)Ścieżka do pola danych, np. system.attributes.dexterity
Tryb (Mode)Jak zmiana jest aplikowana — patrz niżej
Wartość (Value)Liczba lub wartość do zastosowania
PriorytetKolejność aplikowania (niższy = najpierw; domyślnie 20)

Tryby (CHANGE_MODES)

TrybWartośćDziałanie
Custom0Brak automatycznego przetwarzania — dla skryptów niestandardowych
Multiply1Mnożenie: aktualna × wartość
Add2Dodawanie: aktualna + wartość (najczęściej używany)
Downgrade3Zastąp jeśli wartość jest niższa
Upgrade4Zastąp jeśli wartość jest wyższa
Override5Bezwzględne zastąpienie wartością

Przełącznik Ręczny/Dropdown

Checkbox "Ręczny klucz" w nagłówku zakładki Changes przełącza między:

  • Dropdown (domyślnie) — lista predefiniowanych kluczy systemowych pogrupowana wg kategorii

Kiedy używać Changes, a kiedy prepareData?
Changes działa przez standardowy mechanizm Foundry i jest przeliczane automatycznie przy każdej zmianie efektu. prepareData daje pełną kontrolę programistyczną i pozwala na warunkowe modyfikacje. Dla prostych bonusów liczbowych — Changes jest wygodniejszy.

MetodaOpis

Kolejność poziomów: D → sD → L → sL → C → sC → K → sK

// Trigger: manual
// Obniż WSZYSTKIE najcięższe rany o 1 poziom
// (np. masz 2×C i 1×L — obie C staną się sL)
const order = ["D", "sD", "L", "sL", "C", "sC", "K", "sK"];

// 1. Pobierz wszystkie aktywne rany
const wounds = this.actor.items.filter(
  i => i.type === "wound" && i.system.isActive
);

if (wounds.length === 0) {
  this.notification("Brak aktywnych ran", "warn");
  return;
}

// 2. Znajdź najwyższy poziom obrażeń
const maxIdx = Math.max(
  ...wounds.map(w => order.indexOf(w.system.damageType))
);

// 3. Obniż WSZYSTKIE rany na tym poziomie
const worst = wounds.filter(
  w => order.indexOf(w.system.damageType) === maxIdx
);

for (const w of worst) {
  await w.reduceLevel(1);
}

this.notification(`Obniżono ${worst.length} ran z ${order[maxIdx]}`, "info");
